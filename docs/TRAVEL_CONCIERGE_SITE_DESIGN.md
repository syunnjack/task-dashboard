# 推し旅AIコンシェルジュ サイト設計

## 1. 目的

韓国・台湾のライブ、チア、スポーツ、コスメ、グルメ、観光、成人限定イベントへ行く利用者に対し、会話から旅行条件を整理し、準備チェック、通知、予約・購入までを一つの画面で完了させる。

## 2. 主要導線

`目的地 → イベント → 日程 → 旅行経験 → 通信 → 撮影機材 → 準備リスト → 通知・比較・予約`

収益導線はホテル、eSIM、海外Wi-Fi、変換プラグ、保険、コスメ、グルメ・観光予約を優先する。推薦には広告表記を付け、既に用意済みの項目には購入提案を出さない。

## 3. 画面

- 左側: 一問一答チャット、回答履歴、選択肢
- 右側: 進捗、個別チェックリスト、比較・予約候補
- モバイル: チャットの下へチェックリストを積む
- 完了状態は端末に保存し、共有アカウント対応は次段階とする

## 4. 質問項目

1. 韓国・台湾
2. チア、ライブ、スポーツ、コスメ、グルメ、観光、成人限定
3. 出発日
4. 初海外か、経験ありか
5. eSIM、海外Wi-Fi、未定
6. カメラ持参の有無

成人限定イベントは18歳以上の確認後だけ扱い、通常の表示・通知設定から分離する。

## 5. 安全設計

- 入国、航空、電池、撮影規定は公式情報の確認項目として提示する
- パスポート番号、カード番号、本人確認書類画像は収集しない
- 医療・入国・航空会社規則をAIだけで断定しない
- 一般、広告、成人限定を表示上・データ上で区別する

## 6. 段階開発

- MVP: 端末内の会話型診断、チェック、収益候補
- Phase 2: 共有API保存、出発日前通知、イベント連携
- Phase 3: 根拠付き生成AI回答、公式情報更新、旅程最適化
- Phase 4: Web・アプリ同期、同行者共有、B2B管理画面
