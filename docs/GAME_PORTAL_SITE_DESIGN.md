# GAME PORTAL JP サイト設計書

## 1. サービス概要

レトロゲームから最新ゲームまでを対象に、攻略情報と全国のゲーム大会情報を一か所で探せるポータルサイトを構築する。

サービスの二本柱は以下とする。

1. 攻略データベース
   - ゲーム別の基本情報、攻略チャート、ボス攻略、アイテム、キャラクター、対戦テクニック
   - 初心者向け・復帰勢向け・上級者向けを明確に分ける
   - レトロゲームは機種・版・地域差、最新ゲームはアップデート版を管理する
2. 大会ポータル
   - オンライン大会と実店舗・会場大会を横断検索する
   - 都道府県、開催日、ゲーム、参加条件、形式、賞品などで絞り込む
   - 主催者が大会を登録し、運営側が確認・承認できる

仮称は「GAME PORTAL JP」。正式名称とドメインはブランディング段階で決定する。

## 2. サービスコンセプト

### コアメッセージ

「遊び方が見つかる。戦う場所が見つかる。」

### 提供価値

- 昔のゲームも新作も、同じ検索体験で攻略情報へ到達できる
- 大会の開催場所・日時・参加方法を迷わず比較できる
- 店舗や小規模コミュニティも大会を告知できる
- 攻略情報から大会参加へ、大会から練習用攻略へ相互に移動できる
- 更新日・対応バージョン・情報源を表示し、古い情報を見分けられる

### 差別化方針

- 最新タイトル偏重ではなく、レトロゲームを正式な一カテゴリとして扱う
- 攻略メディアと大会カレンダーをゲームIDで統合する
- オンラインと実店舗を同じ検索画面で比較する
- 大会情報に「主催者確認済み」「公式情報確認済み」「ユーザー投稿」を表示する
- ゲームの版、機種、アップデートごとに攻略情報を管理する

## 5. 主要画面設計

### 5.1 ホーム

第一画面は「ゲーム名・大会名・地域」を検索できる統合検索を中心にする。

表示順序：

1. 統合検索
2. 今週・今月の大会
3. 現在地または選択地域の大会
4. 人気ゲーム・新着ゲーム
5. レトロゲーム特集
6. 新着攻略・更新攻略
7. オンライン大会
8. 初参加向け大会ガイド
9. 主催者向け大会登録導線

### 5.2 ゲーム一覧

絞り込み：

- タイトル名
- 発売年代
- プラットフォーム
- ジャンル
- シリーズ
- メーカー
- プレイ人数
- オンライン対応
- 大会あり
- 攻略充実度

カードにはタイトル、機種、発売年、ジャンル、攻略数、今後の大会数を表示する。

### 5.3 ゲーム詳細

ゲームを攻略と大会の共通ハブにする。

- 基本情報
- 対応プラットフォーム・版
- 初心者向け攻略
- 攻略カテゴリ
- 最新更新攻略
- 開催予定大会
- 過去大会結果
- 関連ゲーム・シリーズ
- 更新履歴

レトロゲームでは移植版・リメイク版・地域版の違いを明示する。ライブサービス型ゲームでは対応パッチを表示する。

### 5.4 攻略記事

- 対象ゲーム・機種・版・パッチ
- 難易度と対象レベル
- 結論・要点
- 目次
- 手順本文
- マップ・表・チェックリスト
- 注意点、取り返しのつかない要素
- 出典・検証方法
- 著者・監修者
- 公開日・更新日・変更履歴
- 関連攻略
- 関連大会
- 誤情報の報告

ネタバレ部分は明示し、ユーザーが開くまで隠せるようにする。

### 5.5 大会検索

表示形式はリスト、カレンダー、地図の3種類とする。

絞り込み：

- ゲーム
- オンライン／実店舗／ハイブリッド
- 都道府県・市区町村
- 現在地からの距離
- 開催日・曜日・時間帯
- 初心者歓迎
- 年齢制限
- 参加費
- チーム／個人
- 使用プラットフォーム
- 公認／非公認
- 受付中／残りわずか／締切
- バリアフリー・親子参加

### 5.6 大会詳細

- 大会名・ゲーム名
- 開催日時・受付期限
- オンライン／会場情報
- 地図、最寄り駅、駐車場
- 参加費、定員、参加条件
- 大会形式、ルール、使用機種
- 必要な持ち物
- 賞品・参加特典
- 配信URL
- 主催者プロフィール
- 公式申込URL
- キャンセル・問い合わせ情報
- 情報確認ステータス
- 更新履歴
- 同ゲームの攻略リンク
- 同地域・同ゲームの関連大会

### 5.7 大会登録

入力を5段階に分ける。

1. 基本情報
2. 日時・会場・オンライン環境
3. ルール・参加条件
4. 申込・主催者情報
5. プレビュー・申請

下書き保存、複製、定期開催、開催中止、満員表示に対応する。

### 5.8 口コミ・レビュー

ゲーム、実店舗、会場、大会を対象にレビューできる。

ゲームレビューの評価軸：

- 総合評価
- ゲーム性
- 操作性
- ストーリー
- グラフィック・演出
- 音楽・サウンド
- ボリューム
- 初心者へのおすすめ度

実店舗・会場レビューの評価軸：

- 総合評価
- スタッフ対応
- 設備・プレイ環境
- 大会運営
- アクセス
- 初心者の参加しやすさ
- 清潔さ
- コストパフォーマンス

レビューには、評価、本文、プレイ機種・版、来店・参加時期、ネタバレ、画像、良かった点、改善してほしい点を持たせる。ゲーム詳細、店舗詳細、大会詳細に評価分布と代表レビューを表示する。

信頼性を高めるため、「大会参加確認済み」「店舗チェックイン確認済み」「購入・プレイ情報入力済み」「主催者からの返信」を表示できるようにする。ただし未確認ユーザーも投稿可能とし、確認済みレビューだけでランキングを独占しない補正を行う。

### 5.9 投票

編集部、主催者、ユーザーがテーマ投票を作成できる。ユーザー作成は審査制とする。

投票例：

- いま遊びたいレトロゲーム
- 続編を期待するゲーム
- 初心者におすすめの対戦ゲーム
- 今月のベスト大会
- 好きなキャラクター・ボス・ステージ
- 次に攻略してほしいタイトル

投票には開催期間、単一／複数選択、対象ゲーム、選択肢、画像、説明、結果公開条件を設定する。投票後に結果、地域別・年代別などの集計、関連攻略・大会を表示する。個人を特定できる細かい属性集計は公開しない。

### 5.10 ランキング

ランキングは単純なPV順や平均点順にせず、集計根拠と期間を明示する。

人気ゲームランキング：

- 急上昇：直近の閲覧、検索、保存、レビュー、投票の増加率
- 総合人気：一定期間のユニーク閲覧、保存、レビュー、攻略利用、大会関心
- レトロ人気：発売年代ごとの人気
- 対戦人気：大会・対戦攻略・参加意向を重視
- ユーザー評価：レビューのベイズ補正評価

実店舗人気ランキング：

- 都道府県・地方別
- 大会開催数
- 初心者歓迎度
- 設備・プレイ環境
- 大会参加者の評価
- リピーター・お気に入り数

すべてのランキングに集計期間、更新時刻、対象件数、主な評価要素、広告掲載との非連動を表示する。新規店舗やレビュー件数が少ない対象が不当に不利にならないよう、最低件数とベイズ補正を使用する。

### 5.11 UGCプロフィール

ユーザープロフィールに以下を表示する。

- 好きなゲーム・機種・ジャンル
- 投稿したレビュー、攻略、投票
- 役立った票、採用された訂正
- 大会参加・主催履歴（公開を選択した場合）
- 専門バッジ
- フォロー・フォロワー

本名、正確な住所、未成年者の学校など、不要な個人情報は表示しない。

## 6. 検索・推薦設計

### 統合検索

同じ検索欄から以下を横断する。

- ゲーム
- 攻略
- 大会
- 主催者
- 特集記事

入力補完では、正式名称、略称、旧称、シリーズ名、機種名を扱う。

### 大会推薦

- お気に入りゲーム
- 居住地域・移動可能距離
- オンライン参加可否
- 過去に閲覧・保存した大会
- 初心者向けなどの参加属性

推薦理由を「お気に入りゲーム」「大阪府で開催」「オンライン参加可」のように表示する。

## 8. UGC活性化設計

### UGC循環

```text
ゲーム・大会を発見
  → 攻略・レビューを閲覧
  → 役立った・投票・お気に入り
  → 自分もレビュー・情報を投稿
  → 反応・バッジ・ランキング掲載
  → 再訪・次の投稿
```

### 投稿を増やす施策

- 大会終了後に参加者へレビュー依頼を通知する
- 攻略閲覧後に「この情報は役立ちましたか」を1タップで聞く
- レビュー投稿フォームを評価→短文→詳細の段階式にする
- 「次に攻略してほしいゲーム」を投票で募集する
- レビューが少ないゲーム・地域を募集テーマとして表示する
- 月間投稿テーマ、レトロゲーム発掘月間、地域大会応援企画を開催する
- 下書き保存と投稿テンプレートを提供する
- 初投稿では長文を要求せず、後から追記できるようにする

### 貢献へのフィードバック

- 役立った票と閲覧数を投稿者へ通知する
- 採用された訂正、確認済みレビュー、良質な大会報告へバッジを付与する
- ゲーム別・地域別の貢献者を紹介する
- 編集部おすすめレビュー・攻略を選出する
- 投稿内容がランキングや記事改善に反映された場合に通知する
- 投稿数ではなく、役立ち度・正確性・継続更新を評価する

### バッジ例

- 初レビュー
- 10人の役に立った
- レトロゲーム研究家
- 大会レポーター
- 地域ゲームショップ応援団
- 情報更新協力者
- 攻略監修者
- 認証済み主催者

バッジは金銭価値を持たせず、投稿の正確性や信頼性を保証する表示と混同しない。

### ランキングの不正対策

- 1アカウント・1端末・一定期間の投票制限
- メール確認、レート制限、Bot判定
- 短期間の大量投票・同一ネットワーク・新規アカウント偏重を検知
- 不自然なレビュー文面、相互レビュー、店舗関係者の未申告投稿を検知
- 削除ではなく隔離して運営審査できるようにする
- ランキング計算から疑わしい反応を除外する
- 集計アルゴリズムのバージョンと変更履歴を保存する
- 店舗・メーカー・主催者による組織投票を禁止する

### レビューの公正性

- 店舗・主催者はレビューへ公開返信できるが編集・削除はできない
- 利害関係者の投稿には関係性の申告を必須にする
- 批判的レビューもガイドラインを満たす限り掲載する
- 暴言、個人攻撃、虚偽、個人情報、権利侵害は審査対象にする
- 評価分布、レビュー件数、確認済み比率を併記する
- 平均評価はベイズ補正し、少数の極端な評価による順位操作を防ぐ

### コミュニティガイドライン

- ゲームや店舗への評価と、個人への攻撃を分ける
- ネタバレを適切に設定する
- 他者の攻略・画像・動画を無断転載しない
- 金銭・特典を受けたレビューは明示する
- 未成年者の個人情報を投稿しない
- 大会の勝敗・違反行為に関する断定は根拠を提示する

### UGCモデレーション

1. 投稿時の自動チェック
2. リスクスコアによる公開・保留振り分け
3. ユーザー通報
4. 運営レビュー
5. 投稿者への理由通知と修正機会
6. 異議申し立て
7. 再発時の段階的制限

緊急性の高い個人情報、脅迫、権利侵害は即時非公開とする。通常の低評価や異論は削除理由にしない。

## 9. 大会情報の収集・審査

### 情報経路

1. 主催者による直接登録
2. 公式大会サイト・メーカー情報から編集部登録
3. 店舗・コミュニティの公開情報から編集部登録
4. ユーザーからの情報提供
5. 将来的な提携先フィード・API

### 信頼ステータス

- 公式主催者確認済み
- 公式URL確認済み
- 編集部確認済み
- ユーザー投稿・未確認
- 情報更新待ち
- 中止・終了

大会参加の最終判断は主催者の公式情報で行うよう、すべての詳細ページに注意書きを表示する。

### 更新運用

- 開催7日前、前日、当日に自動再確認タスクを作る
- 受付期限切れ、満員、中止を即時反映する
- 終了後は結果・配信アーカイブを追記する
- 長期間未確認の大会には警告を表示する

## 10. 攻略コンテンツの品質管理

- 対応バージョンを必須表示
- 著者と検証者を分けられる
- 重要な数値はゲーム内検証または公式情報へ紐付ける
- 更新差分を変更履歴として残す
- 誤情報報告を受け付ける
- AI生成文は編集者がゲーム内確認してから公開する
- 無断転載を禁止し、画像・動画・引用の権利を管理する
- レトロゲームは実機・移植版・エミュレーション環境の差を明記する

## 11. SEO・LLMO設計

- Game、VideoGame、Article、Event、Organization、Place、BreadcrumbList等の構造化データを表示内容と一致させる
- ゲーム、攻略、大会、地域を恒久URLで管理する
- 大会終了後も結果ページとして価値を残す
- 都道府県ページは薄い自動生成ページにせず、開催予定・会場・地域ガイドを含める
- 攻略記事は質問見出し、結論先出し、手順、FAQ、対応版、著者、更新履歴を備える
- 主催者・会場・ゲームの表記を統一する
- Review、AggregateRating、ItemList等は検索サービスの要件と表示内容が一致する場合だけ使用する
- ユーザー投稿ページは低品質・重複・スパムを審査し、インデックス対象を制御する
- XMLサイトマップをゲーム、攻略、大会、記事に分ける
- 終了・中止・重複大会のcanonical、noindex、リダイレクト方針を決める

- 保存した攻略の更新通知

- フォロー中ユーザーの新着投稿
- 投票開始・終了・結果発表
- お気に入り店舗の順位・大会更新

通知手段はメールを初期実装し、Web Pushとアプリ通知は後期フェーズで追加する。

## 13. 収益化

優先順位：

1. 大会・店舗の注目掲載枠
2. ゲーム・周辺機器・店舗の広告
3. アフィリエイト
4. 主催者向け有料管理機能
5. スポンサー大会・特集
6. データ提供・API

広告と編集記事を明確に区別し、ランキングやおすすめを広告購入で不正に変更しない。
レビュー投稿や投票の見返りとして、評価内容を条件に報酬を提供しない。投稿キャンペーンを行う場合は、肯定・否定を問わず参加できる条件とスポンサー表示を設ける。

## 15. 非機能要件

- モバイルファースト
- Core Web Vitalsを意識した高速表示
- WCAG 2.2 AAを目標
- 地図を使えないユーザーにも住所・交通情報を提供
- 日本語を初期対象とし、多言語化可能なデータ構造にする
- 時刻はタイムゾーンを保持し、日本時間で明示する
- 個人情報を最小限にし、主催者の私的連絡先を公開しない
- 大会申込・決済は初期段階では主催者公式サイトへ遷移させる
- 投稿・編集・承認の監査ログを保持する
- 投票・リアクションは冪等性を確保し、多重送信を防ぐ
- UGCの削除、非公開、異議申し立て履歴を保持する

## 21. 推奨する初期方針

- 攻略は編集部制作を中心にし、投稿は審査制で段階的に開放する
- 大会申込と決済は主催者公式ページへ任せ、情報ポータルに集中する
- 最新ゲームだけでなく、対戦・RTA・スコア競技が活発なレトロゲームを重点採用する
- 全国対応を掲げつつ、初期は協力主催者を確保できる地域から情報密度を高める
- 最初の成功指標はPVではなく「確認済み大会数」「申込先クリック」「情報更新率」とする
- UGCはレビュー、役立った票、編集部投票から始め、自由投稿コミュニティは後から開放する
- ランキングは週次更新とし、集計方法・件数・期間を公開する
- 実店舗ランキングは全国総合より都道府県別を優先し、地方の店舗にも発見機会を作る
- 投稿数ではなく正確性・役立ち度・継続更新を評価する
